# 90-Day Outbound Execution Plan — First 10–15 Customers

## Outcomes to hit
- **Meetings held:** 45–60
- **Qualified opportunities (SQLs):** 18–24
- **Pilots started:** 12–15
- **Customers closed-won:** 10–15

(Assumes: 30–35% win rate on qualified opps; 45–75 day cycle.)

---

## Week-by-week plan

### Weeks 1–2 — Build list + launch
- Build/enrich **first 250 accounts** (120 Tier A)
- Add **2–3 contacts/account**
- Launch sequences (Email + LI + calls)
- Book first **6–10 meetings**

**Daily targets:**
- Enroll 10–15 new accounts/day
- 40–60 targeted emails/day
- 20–30 calls/day

### Weeks 3–4 — First ROI assessments + pilot design
- Convert meetings into ROI assessments
- Run **8–12 meetings/week**
- Move 6–8 accounts into pilot-scoping

**Assets-first rule:** prioritize accounts with clear packaging/refrigeration + sensor availability.

### Weeks 5–8 — Pilots live + keep top-of-funnel full
- Start **6–10 pilots**
- Continue outbound to reach **500 accounts**
- Weekly pilot reviews; start capturing ROI proof

### Weeks 9–12 — Convert pilots → annual closes
- Pilot success reviews + proposals
- Multi-thread CFO/owner with ROI summary
- Close **10–15 annuals**; line up expansion paths

---

## Funnel math (working backwards)
Target **12 closes**.
- If win rate on qualified opps is 33% → need **36 qualified opps** (aggressive) or improve qualification.
- With tight ICP + pilots, target **50% pilot→close** and **70% meeting→pilot**.

Recommended operating targets:
- **Accounts enrolled:** 500
- **Meetings held:** 50
- **Pilots started:** 15
- **Closes:** 10–15

---

## Conversion tracking (weekly)
Track these rates:
1) Accounts enrolled → positive replies
2) Positive replies → meetings booked
3) Meetings booked → meetings held
4) Meetings held → qualified
5) Qualified → pilot proposed
6) Pilot proposed → pilot started
7) Pilot started → closed-won

---

## Weekly review agenda (30 minutes)
- What copy/angles are winning (downtime, compliance, labor)?
- Which industries within F&B respond best (dairy vs bakery)?
- Where deals are stuck (data access, buyer, timing)?
- Refresh Tier A list (swap in new trigger accounts)

---

## “Do not do” list (protect speed)
- Don’t over-customize first touches (1 sentence only).
- Don’t run pilots without written success criteria.
- Don’t wait for perfect data—start with assets that already have sensors.
- Don’t single-thread (always add Maintenance + Finance as deal progresses).
